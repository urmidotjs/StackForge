Introduction to MongoDB

MongoDB is a popular NoSQL database known for its flexibility and scalability. Unlike traditional relational databases, MongoDB stores data in BSON format, which is a binary version of JSON. It's ideal for applications that deal with large amounts of unstructured or semi-structured data.

Basic Concepts

  • Documents: MongoDB stores data as documents (in BSON format), which can contain complex structures like arrays and embedded documents.
  • Collections: Documents are grouped together in collections, similar to tables in relational databases.
  • CRUD Operations: MongoDB supports Create, Read, Update, and Delete operations through simple queries.

Setting up MongoDB

To start using MongoDB locally, download it from the MongoDB website. Once installed, use MongoDB's shell or connect via a Node.js application.

MongoDB Example

Here’s a basic example of using MongoDB in a Node.js application:


const MongoClient = require('mongodb').MongoClient;
const url = 'mongodb://localhost:27017';
const dbName = 'myDatabase';
MongoClient.connect(url, (err, client) => {
    if (err) throw err;
    const db = client.db(dbName);
    const collection = db.collection('myCollection');
    collection.insertOne({ name: 'John Doe' }, (err, res) => {
        if (err) throw err;
        console.log('Document inserted');
        client.close();
    });
});

Advanced Concepts

  • Indexes: MongoDB supports indexes to improve query performance.
  • Aggregation Framework: MongoDB’s aggregation framework is a powerful tool for data processing and transformation.
  • Sharding: MongoDB supports horizontal scaling through sharding, which allows distributing data across multiple servers.
